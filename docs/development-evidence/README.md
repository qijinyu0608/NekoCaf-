# NekoCafé 开发过程证据目录

本目录用于记录 NekoCafé 平台按 PRD 迭代开发的全过程证据。后续每完成一个可验收版本，都必须在这里新增或更新对应版本记录，并在 Git 提交后推送到远端仓库：

`git@github.com:qijinyu0608/NekoCaf-.git`

## 记录原则

1. 先对齐 PRD，再开发。
2. 每个版本都要说明需求来源、实现范围、关键改动、验证命令和结果。
3. 每个版本至少保留一份版本记录，路径为 `versions/vX.Y.md`。
4. 验证结果必须来自真实命令或真实浏览器检查，不写“应该通过”。
5. 每个版本完成后必须执行 Git 提交、打版本 tag，并推送分支和 tag 到远端。
6. 后续实验四补测试与质量闭环时，应把测试报告、截图、缺陷记录和 RTM 回填也放入本目录或其子目录。

## 目录结构

- `README.md`：本说明。
- `versions/`：每个开发版本的证据记录。
- `artifacts/`：后续放截图、测试报告、录屏脚本、导出日志等大件证据。
