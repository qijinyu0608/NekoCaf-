# D3-8 演示视频脚本

## 1. 演示目标

本次演示聚焦实验三当前已经真实完成的部分，不展示尚未落地的完整 CD、真实 K8s 集群发布或全量 Grafana 平台。

演示目标：

1. 展示 `member-service` 与 `reservation-service` 已可独立运行
2. 展示最小业务闭环可以走通
3. 展示 Docker Compose、CI、Helm、Prometheus 这些工程骨架已经落地

## 2. 演示前准备

建议在演示前完成以下准备：

1. 进入 `project/`
2. 执行 `.venv/bin/pytest -q`
3. 执行 `docker compose up --build -d`
4. 确认以下地址可访问：
   - `http://127.0.0.1:8001/docs`
   - `http://127.0.0.1:8002/docs`
   - `http://127.0.0.1:9090`

## 3. 演示脚本

### 片段一：项目结构与范围说明

讲解词建议：

“本项目是实验二到实验四共用的 Monorepo。在实验三阶段，我们先聚焦两个核心服务，也就是会员服务和预约服务，优先完成本地可运行、可测试、可容器化的最小闭环。”

展示内容：

1. `project/README.md`
2. `project/docs/runbook.md`
3. `project/services/member/`
4. `project/services/reservation/`

### 片段二：本地测试与服务启动

讲解词建议：

“这里先用单元测试保证当前 PoC 是可验证的，然后通过 Docker Compose 把两个服务和 Prometheus 一起启动起来。”

展示命令：

```bash
.venv/bin/pytest -q
docker compose up --build -d
docker compose ps
```

预期说明：

1. 单元测试通过
2. `member` 与 `reservation` 为 `healthy`
3. `prometheus` 处于 `Up`

### 片段三：最小业务链路演示

讲解词建议：

“实验三当前最核心的价值，是把实验二的契约和服务边界真正落成一条可调用的业务链路。这里直接用脚本完成一次从查会员到创建预约再到查预约详情的闭环。”

展示命令：

```bash
make demo-flow
```

预期说明：

1. 查询会员详情成功
2. 查询积分账户成功
3. 查询时段成功
4. 创建预约成功
5. 查询预约详情成功
6. 查询会员预约列表成功

### 片段四：接口与可观测性展示

讲解词建议：

“除了业务接口本身，本轮还补了最小可观测性基线，包括 traceId、请求日志、Prometheus metrics 和基础告警规则。”

展示内容：

1. `http://127.0.0.1:8001/docs`
2. `http://127.0.0.1:8002/docs`
3. `curl http://127.0.0.1:8001/metrics | head`
4. `curl http://127.0.0.1:8002/metrics | head`
5. `http://127.0.0.1:9090/targets`

预期说明：

1. 两个服务都提供 Swagger UI
2. `/metrics` 已输出请求数和请求耗时
3. Prometheus target 显示两个服务均为 `up`

### 片段五：工程化骨架说明

讲解词建议：

“在功能闭环之外，这个仓库还同步落了最小 CI、Helm Chart 和回滚手册，为实验三后续阶段继续补 CD 和最终交付件打基础。”

展示内容：

1. `project/.github/workflows/ci.yml`
2. `project/infra/helm/`
3. `project/docs/rollback.md`
4. `docs/lab3/submission/README.md`
