# NekoCafé 智慧预约首页 PRD

版本：v1  
日期：2026-05-11  
适用范围：`project/` 单体工程实现、实验三 D3-2 源代码仓库、实验四测试对象

## 1. 产品目标

NekoCafé 首屏不再是“演示系统入口页”，而是一个真实品牌预约首页。用户打开页面时，首先感知到的是预约、会员、猫咪互动与推荐，而不是后台结构或技术拆分。

本轮目标：

1. 让顾客在首页完成预约主流程。
2. 让会员积分、猫咪档案、智能推荐成为真实首页模块。
3. 保留一个最小店员后台，处理今日预约和到店确认。
4. 用更简单的单体结构支撑实验三和实验四继续迭代。

## 2. 用户与场景

### 2.1 顾客

顾客最关心的是：

- 今天去哪家店
- 什么时候有位
- 几个人去
- 更适合哪种互动区
- 我有哪些会员权益
- 哪只猫更适合我这次到店体验

对应页面：

- `/`
- `/member`
- `/cats`
- `/recommendations`

### 2.2 店员

店员只需要一个轻量后台：

- 查看今日预约
- 按状态筛选
- 确认到店

对应页面：

- `/staff`

## 3. 首页结构

### 3.1 顶部导航

- 品牌名称：`NekoCafé`
- 固定城市：上海
- 导航项：
  - 立即预约
  - 会员中心
  - 猫咪档案
  - 智能推荐
- 右侧用户区：
  - 未登录时显示体验会员入口
  - 已登录时显示当前 persona 和退出按钮

### 3.2 Hero 区

左侧：

- 品牌主文案
- 价值描述
- 指向会员中心和猫咪档案的入口

右侧：

- 预约表单
- 选择城市
- 选择门店
- 选择日期
- 选择人数
- 选择猫咪互动区
- 可预约时段列表
- 立即预约按钮

### 3.3 首页三张真实数据卡

1. 会员积分卡
   - 当前积分
   - 成长值
   - 可用优惠券数量
2. 我的猫咪档案卡
   - 猫咪昵称
   - 英文名
   - 年龄
   - 品种
   - 性格标签
3. 智能推荐卡
   - 推荐门店
   - 推荐理由
   - 标签摘要

## 4. 功能定义

### 4.1 立即预约

- 城市固定上海
- 门店支持真实切换
- 预约时段按门店、日期、人数查询
- 顾客提交预约时自动建立或复用顾客会话
- 预约成功后进入会员中心查看记录

### 4.2 会员中心

- 展示会员昵称、脱敏手机号、等级
- 展示积分、待入账积分、优惠券数量
- 展示会员权益摘要
- 展示我的预约记录

### 4.3 猫咪档案

- 展示会员关联猫咪资料
- 展示年龄、品种、性格标签
- 展示适合的陪伴风格

### 4.4 智能推荐

- 展示推荐门店
- 展示推荐理由
- 展示标签化原因
- 首版采用规则型推荐，不引入复杂算法服务

### 4.5 店员后台

- 店员 persona 独立进入
- 查看指定门店、指定日期的预约
- 按状态筛选
- 执行到店确认

## 5. 技术实现约束

- 技术栈：`FastAPI + Jinja + 原生 JS + SQLite`
- 工程形态：全栈单体
- 不再依赖独立 React/Vite 前端作为主实现
- SQLite 继续作为课程 PoC 的唯一数据库
- 会话采用 mock persona：
  - `customer -> Momo`
  - `staff -> Aki`
- 首页先不依赖 logo 资源，先保证版式与信息结构成立

## 7. 验收标准

### 页面验收

- 首页首屏具备真实预约站的视觉重心
- 首屏同时看到预约、积分、猫咪档案、推荐
- `/staff` 不再复用顾客首页的视觉结构

### 功能验收

- 顾客可完成预约闭环
- 会员中心可看到预约记录
- 猫咪档案与推荐页展示真实数据
- 店员后台可看到今日预约并确认到店

### 工程验收

- 项目主实现为单体应用
- `project/frontend` 不再作为主实现存在
- 课程实验相关文档目录保留完整
