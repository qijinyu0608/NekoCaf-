# NekoCafé 智慧预约平台 PRD

版本：v1  
日期：2026-05-11  
适用范围：`project/` 单体工程实现、实验三 D3-2 源代码仓库、实验四测试对象

## 1. 产品目标

NekoCafé 首屏不再是“演示系统入口页”，而是一个真实品牌预约首页。用户打开页面时，首先感知到的是预约、会员、猫咪互动与推荐，而不是后台结构或技术拆分。

本轮目标：

1. 让顾客在首页完成预约主流程。
2. 让会员积分、猫咪档案、智能推荐成为真实首页模块。
3. 保留一个最小店员后台，处理今日预约和到店确认。
4. 支持多城市门店探索，避免产品体验只局限于上海。
5. 增加预约确认页、门店可信信息与到店须知，让流程更像成熟商业产品。
6. 用更简单的单体结构支撑实验三和实验四继续迭代。

## 2. 用户与场景

### 2.1 顾客

顾客最关心的是：

- 今天去哪家店
- 什么时候有位
- 几个人去
- 更适合哪种互动区
- 我有哪些会员权益
- 哪只猫更适合我这次到店体验
- 预约成功后如何确认到店信息

对应页面：

- `/`
- `/stores`
- `/reservations/{reservation_id}`
- `/member`
- `/cats`
- `/recommendations`

### 2.2 店员

店员只需要一个轻量后台：

- 查看今日预约
- 按状态筛选
- 确认到店
- 仅访问自己门店范围内的数据

对应页面：

- `/staff`

### 2.3 管理员

管理员只负责课程 PoC 阶段的权限查看与演示：

- 查看 mock persona、角色、权限点
- 查看会员范围与门店范围
- 验证顾客、店员、管理员三类入口已经分离

对应页面：

- `/permissions`

## 3. 首页结构

### 3.1 顶部导航

- 品牌名称：`NekoCafé`
- 固定城市：上海
- 导航项：
  - 立即预约
  - 门店探索
  - 会员中心
  - 猫咪档案
  - 智能推荐
- 右侧用户区：
  - 未登录时显示体验会员入口
  - 已登录时显示当前 persona 和退出按钮

### 3.2 Hero 区

左侧：

- 品牌主文案
- 价值描述
- 指向会员中心和猫咪档案的入口

右侧：

- 预约表单
- 选择城市
- 选择门店
- 选择日期
- 选择人数
- 选择猫咪互动区
- 可预约时段列表
- 立即预约按钮

### 3.3 首页三张真实数据卡

1. 会员积分卡
   - 当前积分
   - 成长值
   - 可用优惠券数量
2. 我的猫咪档案卡
   - 猫咪昵称
   - 英文名
   - 年龄
   - 品种
   - 性格标签
3. 智能推荐卡
   - 推荐门店
   - 推荐理由
   - 标签摘要

## 4. 功能定义

### 4.1 立即预约

- 城市支持上海、杭州、南京、北京、成都等演示门店
- 门店支持真实切换
- 预约时段按门店、日期、人数查询
- 顾客提交预约时自动建立或复用顾客会话
- 预约成功后进入会员中心查看记录

### 4.2 门店探索

- 以城市筛选门店
- 展示每家门店的区域、特点标签、可约时段数量
- 展示地址、营业时间、联系电话等可信信息
- 展示最早可约时间和前两个可约时段预览
- 从门店卡片直接进入预约页并带入门店

### 4.3 预约确认

- 预约成功后进入确认页
- 展示预约号、状态、到店时间、人数、桌位、互动区
- 展示门店地址、营业时间、电话
- 展示到店须知和取消入口
- 仅会员本人可查看自己的预约详情

### 4.4 会员中心

- 展示会员昵称、脱敏手机号、等级
- 展示积分、待入账积分、优惠券数量
- 展示会员权益摘要
- 展示我的预约记录
- 突出下一次到店

### 4.5 猫咪档案

- 展示会员关联猫咪资料
- 展示年龄、品种、性格标签
- 展示适合的陪伴风格

### 4.6 智能推荐

- 展示推荐门店
- 展示推荐理由
- 展示标签化原因
- 首版采用规则型推荐，不引入复杂算法服务

### 4.7 店员后台

- 店员 persona 独立进入
- 查看指定门店、指定日期的预约
- 按状态筛选
- 执行到店确认
- 店员只能读取和核销自己门店范围内的预约

### 4.8 权限管理

- 会话返回角色、权限点和数据范围
- 顾客权限只覆盖会员资料、预约、猫咪档案和推荐
- 店员权限只覆盖门店预约读取和到店确认
- 管理员权限覆盖权限配置查看，不参与顾客预约链路

## 5. 技术实现约束

- 技术栈：`FastAPI + Jinja + 原生 JS + SQLite`
- 工程形态：全栈单体
- 不再依赖独立 React/Vite 前端作为主实现
- SQLite 继续作为课程 PoC 的唯一数据库
- stores 表包含地址、营业时间和门店电话；旧 SQLite 库通过兼容迁移补列
- 会话采用 mock persona：
  - `customer -> Momo`
  - `staff -> Aki`
  - `admin -> Rin`
- 首页先不依赖 logo 资源，先保证版式与信息结构成立

## 7. 验收标准

### 页面验收

- 首页首屏具备真实预约站的视觉重心
- 首屏同时看到预约、积分、猫咪档案、推荐
- `/stores` 支持按城市浏览门店和可约时段摘要
- `/reservations/{reservation_id}` 支持预约成功后的确认页
- `/staff` 不再复用顾客首页的视觉结构
- 页面整体字体、圆角、色块和阴影保持克制统一

### 功能验收

- 顾客可完成预约闭环
- 顾客可从门店探索页进入指定门店预约
- 顾客预约成功后可查看预约确认、到店须知与取消入口
- 会员中心可看到预约记录
- 猫咪档案与推荐页展示真实数据
- 店员后台可看到今日预约并确认到店

### 工程验收

- 项目主实现为单体应用
- `project/frontend` 不再作为主实现存在
- 课程实验相关文档目录保留完整
