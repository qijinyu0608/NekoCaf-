# NekoCafé v1 开发说明

## 本轮重构目标

把原先的“双服务 + React 演示壳”收束成一个更容易继续长大的真实产品雏形：

- 首页就是预约站
- 会员、猫咪、推荐是首页真实模块
- 店员后台最小保留
- 工程实现改为 FastAPI 全栈单体

## 为什么这样改

旧结构的主要问题不是“技术不对”，而是“复杂度已经超过当前产品阶段”：

- 首页不像首页，更像演示页
- 顾客端和后台虽然分壳，但仍显得重
- 目录和运行入口过多，不利于继续做实验三和实验四

## 当前实现边界

- 保留课程实验相关 docs
- 删除开发过程中的旧截图证据和前端试验痕迹
- SQLite 继续保留
- 推荐能力先用规则型结果
- 猫咪档案先做展示，不做编辑
