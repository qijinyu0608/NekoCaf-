# 0004 采用事件总线解耦通知、推荐与审计

- Status: Accepted
- Topic: 消息中间件

## Context
预约主链路不应被通知、推荐解释、运营分析等扩展能力拖慢。

## Decision
引入事件总线作为异步解耦基础设施，预约与会员服务只负责发布领域事件，由通知、推荐、审计等消费者异步处理。

## Consequences
- 正面：降低主链路同步耦合，便于后续扩展分析与推荐。
- 负面：增加消息幂等、重试和监控的复杂度。
- 中性：实验三可先用轻量实现模拟总线，再逐步增强。

## Alternatives Considered
- 全部同步 HTTP 调用：主链路容易被扩展服务拖慢。
- 完全不引入异步：难以体现架构演进空间。
