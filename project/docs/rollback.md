# Rollback Guide

## 当前适用阶段

当前文档适用于实验三早期阶段，也就是“本地功能先落地、流水线尚未完全接入”的阶段。

## 当前回滚策略

当前回滚不依赖发布平台，主要采用源码和本地运行层面的简单回退：

1. 保留 `docs/foundation/` 的上一版导出件。
2. 对重要表格采用版本号递增，如 `v0.1`、`v0.2`。
3. 服务代码若引入回归，优先回退到最近一个通过本地测试的 Git 提交。
4. 若 Docker 或 Compose 配置导致本地不可运行，优先回退到最近一个可启动版本，再重新补增量改动。

## 实验三阶段性升级方向

在实验三后续阶段，回滚策略会逐步升级为：

1. Git 提交级回滚
2. Compose 配置级回滚
3. Helm values 或镜像标签级回滚
4. 流水线触发后的标准化回滚脚本

## 当前限制

当前阶段尚未接入：

- 自动回滚
- 金丝雀流量切分
- 基于监控阈值的发布回退

这些能力将在实验三后续阶段补齐，而不是在功能尚未稳定时提前构建。
